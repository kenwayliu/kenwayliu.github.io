CloudStack

Cloudstack 4.2.0 介绍

一.CloudStack组成部分讲解：

  1. Zone：Zone 对应于现实中的一个数据中心，它是 CloudStack 中最大的一个单元。
  2. Pod：Pod 对应着一个机架。同一个 pod 中的机器在同一个子网（网段）中。
  3. Cluster：Cluster 是多个主机组成的一个集群。同一个 cluster 中的主机有相同的硬件，相同的 Hypervisor，和共用同样的存储。同一个 cluster 中的虚拟机，可以实现无中断服务地从一个主机迁移到另外一个上。
  4. Host：Host 就是运行虚拟机（VM）的主机。
  5. 即从包含关系上来说，一个 zone 包含多个 pod，一个 pod 包含多个 cluster，一个 cluster 包含多个 host。
  6. CloudStack 中存在两种存储：
  7. Primary storage：一级存储与 cluster 关联，它为该 cluster 中的主机的全部虚拟机提供磁盘卷。一个 cluster 至少有一个一级存储，且在部署时位置要临近主机以提供高性能。
  8. Secondary storage：二级存储与 zone 关联，它存储模板文件，ISO 镜像和磁盘卷快照。
  9. 模板：可以启动虚拟机的操作系统镜像，也包括了诸如已安装应用的其余配置信息。
  10. ISO 镜像：包含操作系统数据或启动媒质的磁盘镜像。
  11. 磁盘卷快照：虚拟机数据的已储存副本，能用于数据恢复或者创建新模板

Shell

Shell编程基础

1.shell 编程基础

  • shell:是一种解释器,提供人与机器交流的接口.
  • 为社么要用shell
  • 1.解决重复操作的事情
  • 2.节约时间,提高工作效率
  • 3.功能强大,使用简单.
  • 查看系统注册的shell

2.shell的基本组成元素

  • 1.#!:表示魔法字符或者沙帮,出现在脚本的第一行,用于定义命令解释器
  • 2.#:以#开头的行表示注释,这些不被运行,只是给人阅读使用.
  • 3.系统命令:shell脚本中运行解释的系统命令
  • 4.变量:脚本运行过程中某些反复调用的值的暂存池
  • 5.流程控制语句:判断,循环,跳转等流程控制.
